# Phase 0 Exit Review

Date: 2026-07-17

Status: Implementation complete

## Delivered walking skeleton

The Go CLI and embedding API now execute this Python-free path:

```text
URL -> shared HTTP transport -> deterministic extractor registry
    -> ordered normalized metadata -> minimal format selection
    -> confined output template -> resumable direct HTTP downloader
    -> atomic final file + structured events + deterministic metadata JSON
```

The fixture extractor and generic direct-media extractor are deliberately the
only extractor claims. HLS, DASH, production YouTube, ffmpeg, authentication,
impersonation, plugins, and full yt-dlp syntax remain Phase 1 or later work.

## Verification performed

- `gofmt` produced no changes.
- `go mod tidy` produced no dependency drift.
- `go test ./...` passed, including an actual CLI subprocess download.
- `go vet ./...` passed.
- `go test -race ./...` passed.
- `FuzzValueJSON` passed a three-second run with more than 300,000 executions.
- `FuzzResolve` passed a three-second run with more than 170,000 executions.
- Static, CGO-free builds succeeded for Linux amd64, Windows amd64, macOS
  amd64, and macOS arm64.
- Docker image `ytdlp-go-python-free:phase0-complete` built successfully. Its
  Alpine build stage proved `python` and `python3` absent, validated all 28
  capabilities, ran the complete test/E2E suite, and produced a static binary
  for a final `scratch` image.

Native Windows execution is assigned to the checked-in GitHub Actions matrix;
there is no Git remote in this local repository on which to observe that hosted
job yet. The local Windows cross-build is complete, and a future remote push is
not allowed to claim a release until the native matrix passes.

## Security review

- Page extraction is response-size bounded.
- Authorization, cookies, proxy authorization, URL user information, and common
  signed query values are redacted from diagnostic helpers.
- Output templates reject absolute paths and parent traversal, sanitize
  cross-platform-invalid names, and confine destinations to the output root.
- Partial state is written with owner-only permissions and final media is moved
  only after completion and sync.
- Downloads use direct HTTP requests, bounded retry count, context cancellation,
  and validated ETag/range resume.
- Tests use generated bytes and contain no external credentials or account data.
- The final runtime image contains the Go executable and no Python runtime or
  shell.

The repository records its current dependency licenses and fixture provenance in
`THIRD_PARTY_NOTICES.md`. Selecting the project's own public-distribution license
is a release-governance decision and remains required before publishing binaries
or source under an open-source license.

## Exit decision

The Phase 0 implementation gate is passed. All Phase 0 entries in the capability
manifest are `compatible` with named evidence, while later capabilities remain
`not_started`. Phase 1 may build on the current metadata, transport, lifecycle,
event, template, and downloader boundaries without replacing them.
